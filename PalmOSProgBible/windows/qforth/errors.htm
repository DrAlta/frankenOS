Common Programming Errors

The following programming errors are commonly encountered when developing a Forth application.  Many of them are caught by the safe.txt module in the library

  • Division by zero
  • Stack overflow -- leaving unconsumed cells on the data stack
  • Stack underflow -- insufficient arguments on the data stack
  • Unmatched IF ELSE THEN
  • Unmatched BEGIN WHILE REPEAT (missing WHILE, or using UNTIL or AGAIN instead of REPEAT)
  • Unmatched DO LOOP
  • Missing or superfluous ; (semi-colon)
  • Return stack imbalances -- unbalanced >R R>
  • Cell-sized reads/writes to unaligned (odd) addresses
  • Attempts to write to storage memory without using DmWrite
  • LEAVE used outside of a DO LOOP
  • Input-buffer overrun (source line longer than 80 characters)
